JDF Folder Watcher

Aanvrager: Antony Geldof Datum: 29 april 2026 Classificatie: Interne infrastructuur

1. Samenvatting

De productieplanningsapplicatie Omni importeert automatisch productieorders vanuit XML-bestanden (JDF-formaat) die door het drukkerijsysteem op een gedeelde netwerkschijf worden geplaatst. Om de belasting op het netwerk en de netwerkschijf te minimaliseren, wordt gevraagd om een lichtgewicht PowerShell-script als geplande taak te laten draaien op de bestandsserver.

Het script bevat geen installatie, geen externe verbindingen en geen rechteningreep. Het schrijft uitsluitend één klein bestand op dezelfde gedeelde schijf.

2. Probleemstelling

Zonder dit script doorzoekt Omni elke 10 minuten de volledige gedeelde map (tot 80.000 bestanden) om te controleren of er nieuwe productieorders zijn. Dit genereert onnodige netwerkbelasting, ook wanneer er geen nieuwe bestanden zijn.

Situatie Netwerkoproepen per dag Opmerking
Zonder watcher (huidig) ~11,5 miljoen 144× volledige directoryscan van 80.000 bestanden
Met watcher ~144 1 klein bestand (trigger) elke 10 minuten gecontroleerd

3. Wat het script doet

  • Wordt elke 10 minuten gestart via de Windows Taakplanner, controleert zijn werk en sluit dan af.
  • Zoekt op de lokale schijf naar DT*.xml-bestanden die gewijzigd zijn sinds de vorige run.
  • Schrijft bij wijzigingen de bestandsnamen naar _jdf_trigger.json op de gedeelde schijf.
  • Omni leest enkel dat kleine bestand en verwerkt alleen de vermelde bestanden.

4. Wat het script niet doet

  • Geen verbinding met het internet of externe systemen.
  • Geen toegang tot andere mappen dan de opgegeven JDF-map.
  • Geen wijzigingen aan het systeem, register of beveiligingsinstellingen.
  • Geen nieuwe software-installatie vereist.

6. Script (volledig, ter review)

Bestand: C:\Scripts\jdf-watcher.ps1

# JDF Folder Watcher
# Wordt elke 10 minuten uitgevoerd via de Windows Taakplanner.
# Zoekt DT*.xml bestanden gewijzigd sinds de vorige run.
# Schrijft gewijzigde bestandsnamen naar _jdf_trigger.json op de gedeelde map.
# Geen externe verbindingen. Geen installatie vereist. Geen persistent proces.

param(
    [string]$WatchPath = "D:\JDF-Share"    # Pas aan naar de juiste map
)

$triggerFile   = Join-Path $WatchPath "_jdf_trigger.json"
$timestampFile = Join-Path $WatchPath "_jdf_lastrun.txt"

# Lees tijdstip van vorige run
$lastRun = if (Test-Path $timestampFile) {
    [datetime]::Parse((Get-Content $timestampFile -Raw).Trim())
} else {
    [datetime]::MinValue
}

# Zoek DT*.xml bestanden gewijzigd na de vorige run
$changed = Get-ChildItem -Path $WatchPath -Filter "DT*.xml" -File |
           Where-Object { $_.LastWriteTime -gt $lastRun } |
           Select-Object -ExpandProperty Name

# Schrijf trigger als er wijzigingen zijn
if ($changed.Count -gt 0) {
    $json = $changed | Sort-Object | ConvertTo-Json -Compress
    Set-Content -Path $triggerFile -Value $json -Encoding UTF8
    Write-Host "Trigger geschreven: $($changed.Count) bestand(en)"
}

# Sla huidig tijdstip op als referentie voor de volgende run
(Get-Date -Format "o") | Set-Content -Path $timestampFile -Encoding UTF8

7. Installatie als geplande taak

Eenmalig uitvoeren in een verhoogde PowerShell-sessie:

$action = New-ScheduledTaskAction `
            -Execute  "powershell.exe" `
            -Argument "-NonInteractive -File C:\Scripts\jdf-watcher.ps1"

$trigger = New-ScheduledTaskTrigger -RepetitionInterval (New-TimeSpan -Minutes 10) `
             -Once -At (Get-Date)

$settings = New-ScheduledTaskSettingsSet `
              -ExecutionTimeLimit (New-TimeSpan -Minutes 5) `
              -StartWhenAvailable $true

Register-ScheduledTask `
  -TaskName   "JDF Folder Watcher" `
  -Action     $action `
  -Trigger    $trigger `
  -Settings   $settings `
  -RunLevel   Limited `
  -Description "Controleert elke 10 min de JDF-map en schrijft een trigger voor Omni."
